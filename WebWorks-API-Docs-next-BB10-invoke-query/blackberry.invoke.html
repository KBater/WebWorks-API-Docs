Invoke

The Invoke object contains methods that interact with other applications on a BlackBerry PlayBook.

The invoke() method on the invoke object allows you to pass arguments to the target application.
The types of arguments can be one of: blackberry.invoke.AddressBookArguments, blackberry.invoke.BrowserArguments, blackberry.invoke.CalendarArguments, blackberry.invoke.CameraArguments, blackberry.invoke.JavaArguments, blackberry.invoke.MapsArguments, blackberry.invoke.MemoArguments, blackberry.invoke.MessageArguments, blackberry.invoke.PhoneArguments, blackberry.invoke.SearchArguments, and blackberry.invoke.TaskArguments.


Supported Platform(s)

- BlackBerry PlayBook 1.0+
- BlackBerry 10

invoke

static void invoke(appType : Number, [args: Object])


Supported Platform(s)

 - BlackBerry OS 5.0+
 - BlackBerry PlayBook 1.0+
 - BlackBerry 10
 - Ripple Emulator

Description

Invokes another application on the BlackBerry Playbook.



Parameter Type Description
appType Number an integer value representing the type of application to launch. Must be one of the 'APP_*' constants.
args Object
Optional 		An arguments object specifying information for the application being invoked.

Throws Description
Exception If values supplied are not correct.

query

static void query(request : Object, onSuccess : function, onError : function)


Supported Platform(s)

 - BlackBerry 10

Description

Queries device for list of invokable applications.



Parameter Type Description
request Object An object containing a query to be performed on applications on the device.

action: An identifier of the action to be performed by the target. Omitting action implies the query should apply to any action supported for the specified type or that the target should infer the action.
type: The MIME type of data to be acted on. It must be provided if the uri attribute is not provided.
uri: Used to infer the MIME type of the data. It must be provided if type is not specified.
target_type: Indicates the types of targets to be returned by the query. Possible values are "VIEWER", "APPLICATION", "SERVICE" or "ALL".
action_type: Indicates the type of actions to be returned. Possible values are "MENU" or "ALL". Menu actions specify addtional icon and label properties.
receiver-capabilites: The list of capabilities that must be granted to the target in order for it to be considered a candidate.
brokering_mod: A modifier describing the querying application's preferences for brokering (e.g. prefer the foreground application if it is a candidate).
onSuccess function The callback to be performed if the query is successful.

response: An array of objects containing information about invocations that can be made based on the request. Each object contains information pertaining to one action. Each array contains action, icon, label, default and targets properties.
response.action: The action that is supported for the specified query.
response.icon: The path of the icon file that may be used to represent the action.
response.label: The label that can be used to represent the action.
response.default: The name of the target that should be considered as the default provider for the action. If the attribute is omitted then there is no default.
response.targets: An array of objects containing information about targets that correspond to the action. Each object has key, icon, splash, label, and type properties.
response.targets.key: An identifier of target that supports the invocation.
response.targets.icon: The path of the icon that may be used to represent the target.
response.targets.splash: The path of the icon that may be used to represent the viewer while loading. This field is mandatory if the type property is VIEWER and is otherwise omitted.
response.targets.label: The label that may be used to represent the target.
response.targets.type: Indication of the type of the invocation target. Possible values are "APPLICATION", "VIEWER" or "SERVICE".
onError function The callback to be performed if there was an error processing the query.

error: A String that contains an error message.

Code Example(s)

<script type="text/javascript">

 function onSuccess(response) {
     alert("response: " + JSON.stringify(response, null, 2);
 }

 function onError(error) {
     alert("error: " + error);
 }

 function performQuery() {
     var request = {
             "action": bb.action.open,
             "type": image/*,
             "target_type": "APPLICATION",
             "action_type": "MENU"
         };

     blackberry.invoke.query(request, onSuccess, onError);
 }

</script>
