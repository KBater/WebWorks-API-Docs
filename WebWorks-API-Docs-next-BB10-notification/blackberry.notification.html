API Reference

notification

The blackberry.notification API allows you to create a notification object for BlackBerry 10 and then you can notify, cancel or close it afterwards.

Constructors

blackberry.notification (message : Object)


Supported Platforms
 - BlackBerry 10


Parameters
message Mandatory. This notification message object supplies the information needed to create various types of notifications.

itemId: Mandatory. Unique id that identifies this notification. Can be referenced in subsequent messages and responses related to this notification.
title: Used as text in toasts (if there is no subtitle text) and Universal inbox entries.
subtitle: Used as text in toasts and Universal inbox entries.
target: Target is used with UIB Transient Notifications if the notification is opened from UIB and also directly if launchInBackground is set to true. This property should match the identifier returned by the invocation framework when the target application registered with the invocation framework. The value none means that there is no launch behaviour for this notification. Value is passed on to the Invocation Framework as target.
targetAction: Action as registered with invocation framework. Value is passed on to the Invocation Framework as action. Example bb.action.PUSH.
payload: Payload to send to the invoked app. Data must be Base64 encoded. Value is passed on to the Invocation Framework as data. Data is take from PayloadURI if specified or no data is delivered on invoke.
payloadURI: URI to payload data to send to the invoked app. Value is passed on to the Invocation Framework as uri. Example: "file://path/to/file". Data is taken from payload parameter, if specified, or no data is delivered on invoke.

Code Example: 
<script type="text/javascript">

  function onSuccess() {
      alert("Notification sent successfully!");
  }

  function onError(error) {
      alert("Sending notification failed, error: " + error);
  }

  // Simple notification with LED, vibrate and sound
  var message = {
	     itemId : "notification1"
  }
  var n = new blackberry.notification(message);
  n.notify(onSuccess, onError);

  // UIB transient notification
  var message = {
	     itemId : "transient1",
      title : "You have a notificaiton",
      subTitle : "Some details"
  }
  var n = new blackberry.notification(message);
  n.notify(onSuccess, onError);

  // User open the application so the UIB notification can be closed
  n.close(onSuccess, onError);

  // To be verified and completed
  // For Push event
  var message = {
	     itemId : "pushevent1",
      title : "",        // e.g. Application Name
      subtitle : "",  // e.g. Received a push notification
      target : "",         // id received from invocation manager
      payload : ""         // optional payload to include, base 64 encoded
  }
  var n = new blackberry.notification(message);
  n.notify(onSuccess, onError);

<\script>

Functions

void cancel ([onSuccess: function()], [onError: function([error: String])])


This method will cancel the notification, which includes removing the new splat for an application and turning off the LED if there are no other notifications for which the LED is flashing.


Supported Platforms
 - BlackBerry 10


Parameters
onSuccess The callback function that will be triggerd if the action is successful.
onError The callback function that will be triggered if there was an error proforming the action.

error: A String that describes the error.

void close ([onSuccess: function()], [onError: function([error: String])])


This method will delete the notification from the UIB notifications area automatically. The user also has the ability to delete notifications from the UIB notifications area manually.


Supported Platforms
 - BlackBerry 10


Parameters
onSuccess The callback function that will be triggerd if the action is successful.
onError The callback function that will be triggered if there was an error proforming the action.

error: A String that describes the error.

void notify ([onSuccess: function()], [onError: function([error: String])])


This method will send the notification based on the information set in current notification object.


Supported Platforms
 - BlackBerry 10


Parameters
onSuccess The callback function that will be triggerd if the action is successful.
onError The callback function that will be triggered if there was an error proforming the action.

error: A String that describes the error.

Properties
