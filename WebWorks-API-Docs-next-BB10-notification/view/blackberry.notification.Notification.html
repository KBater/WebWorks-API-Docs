notification

The blackberry.notification API allows you to create a notification object in UIB for BlackBerry 10.

Constructors

blackberry.notification.Notification (title : String, options : Object)


Supported Platforms
 - BlackBerry 10


Parameters
title Mandatory. Used as text in toasts (if there is no subtitle text) and Universal inbox entries.
options The options object supplies more information needed to create the notification.

body: Used as text in toasts and Universal inbox entries.
tag: Tag of the notification. It can be used for closing the notification by static method blackberry.notification.Notification.remove().
target: Target is used with UIB Transient Notifications if the notification is opened from UIB and also directly if launchInBackground is set to true. This property should match the identifier returned by the invocation framework when the target application registered with the invocation framework. The value none means that there is no launch behaviour for this notification. Value is passed on to the Invocation Framework as target.
targetAction: Action as registered with invocation framework. Value is passed on to the Invocation Framework as action. Example bb.action.PUSH.
payload: Payload to send to the invoked app. Data must be Base64 encoded. Value is passed on to the Invocation Framework as data. Data is take from PayloadURI if specified or no data is delivered on invoke.
payloadURI: URI to payload data to send to the invoked app. Value is passed on to the Invocation Framework as uri. Example: "file://path/to/file". Data is taken from payload parameter, if specified, or no data is delivered on invoke.
payloadType: The MIME type of URI or payload data to send to the invoked app. Value is passed on to the Invocation Framework as type. Example: "text/plain".
onclick: It is ignored for WebWorks appliction. Insteadly, use oninvoked event handler, which will be triggered when the user clicks the notification in UIB.
onshow: It will be triggered if the notification is created successfully; or onerror callback will be triggered.
onerror: It will be triggered if the notification is created unsuccessfully; or onshow callback will be triggered.
onclose: It is ignored for WebWorks appliction as it has no clear usage of it.

Code Example: 
<script type="text/javascript">

  // UIB transient notification
  var title = "You have a notificaiton";
  var options = {
      body : "Some details"
  }
  var n = new blackberry.notification.Notification(title, options);

  // User open the application so the UIB notification can be closed
  n.close();

  // To be verified and completed
  // For Push event
  var title = "";    // e.g. Application Name
  var options = {
      body : "",  // e.g. Received a push notification
      target : "",     // id received from invocation manager
      payload : "",    // optional payload to include, base 64 encoded
      payloadType : ""               // MIME type of payload
  }
  var n = new blackberry.notification.Notification(message);

<\script>

static void blackberry.notification.Notification.remove (tag : String)


This static method will delete the notification from the UIB notifications area automatically. UIB notifications will be removed manually if the user clicks it.
